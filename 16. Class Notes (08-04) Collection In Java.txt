Limitations of an Array:-
=-=-=-=-=-=--=--=-=--=-=
1. Fixed Size:-
- Once an array is created, its size cannot be changed.
- You must know the size in advance.
- Problem: If you need more space, you must create a new array and copy elements.

2. Homogeneous Data
- Arrays can only store elements of the same data type (except Object[] arrays).

3. No Built-in Methods for Manipulation
- Arrays lack rich utility methods like adding, removing, or searching elements.

4. Memory Waste (for Large Arrays)
- If you declare a large array but use only a few elements, memory is wasted.
- Fixed size can lead to under-utilization.







Let's create a Dynamic Array:-
=-=-==-=-=-=-=-=-=-=-=-=-=-=
Code:-

public class DynamicArray {
    Object arr[];
    int size;
    int capacity;
    DynamicArray(){
        capacity =10;
        arr = new Object[capacity];
        size=0;
    }
    DynamicArray(int capacity){
        this.capacity =capacity;
        arr = new Object[capacity];
        size=0;
    }
    void increaseCapacity(){
        int newCapa = capacity+(capacity/2);
        Object[] newArr = new Object[newCapa];
        for(int i=0; i<size; i++){
            newArr[i] = arr[i];
        }
        arr = newArr;
        capacity = newCapa;
    }
    void add(Object ele){
        if(size==capacity){
            increaseCapacity();
        }
        arr[size] = ele;
        size++;
    }
    int searchEle(Object ele){
        for(int i=0; i<size; i++){
            if(arr[i]==ele){
                return i;
            }
        }
        return -1;
    }
    int getSize(){
        return size;
    }
    void display(){
        for(int i=0; i<size;i++){
            System.out.print(arr[i]+" ");
        }
        System.out.println();
    }
}
public class Demo {
    public static void main(String[] args) {
        DynamicArray da = new DynamicArray();
        for(int i=0; i<20; i++){
            da.add(i);
        }

        da.display();
        System.out.println(da.getSize());
        System.out.println(da.searchEle(10));
        System.out.println(da.searchEle(30));
    }
}

Collection :-
=-=-=-=-=-=-=
A Collection in Java is a framework that provides a set of classes and interfaces to store, manage, and manipulate groups of objects dynamically.
							or
A collection is a group of individual data that are grouped together to form a single unit. Collections are containers that hold multiple data items as a single unit.

The two root interfaces of java collection class are:
	- Collection interface(java.util.collection)
	- Map interface(java.util.Map)


Collection Framework in Java:-
=-=-=-=-=-=-=-=-=-=--==-=-=-=
The Java Collection Framework (JCF) is a set of classes and interfaces that provide predefined data structures to store, retrieve, and manipulate groups of objects efficiently. It is part of the java.util package.


Hierarchy of Collection F/W:-
=-=-=-=-=-=-=-=-==-=-=-=-=-=
Collection (Interface)
|
|----> List (Interface)
|
|----> Set (Interface)
|
|----> Queue (Interface)



Basic Functionalities of List, Set, and Queue Interface:-
=-=-=-=-=-=-=-==-=-=-=-=-=-=-==-=--=-=-=-=--=-=-=--=-=-=-=

1. List Interface
------------------
Functionality
- Ordered collection (maintain insertion order)
- Duplicates allowed
- Index-based access (0,1,2…)
- Null values allowed

Use Case
- Order important ho
- When need duplicate values
- Element need access by Index


2. Set Interface
---------------
Functionality
- No duplicates allowed
- Order generally not guaranteed (except some implementations)
- No index-based access
- Null allowed (usually only one)

Use Case
- Need unique elements 
- Avoid duplicate 

3. Queue Interface
------------------
Functionality
- FIFO (First In First Out)
- Elements process in Order
- Mainly used for processing tasks
- No index access

Use Case
- Use in Line/queue system (like ticket counter)
- Need Task scheduling

1. ArrayList:-
=-=-=-=-=-=-=
- ArrayList is a resizable array implementation of the List interface.
- It allows dynamic growth, unlike arrays which have fixed size.
- It maintains insertion order, allows duplicate elements, and nulls.
- Implements:
	- List<E>
	- Cloneable
	- Serializable
- RandomAccess (supports fast index-based access)

Internal Storage of ArrayList
-----------------------
- Internally, ArrayList uses a dynamic array, specifically an Object[] array called elementData.
- This array is not exposed directly; all operations like add, get, remove work through methods.

Default Size and Capacity
--------------------
Default initial capacity (Java 8+): 10

- ArrayList<Integer> list = new ArrayList<>(); // capacity = 10

If you provide an initial capacity:

- ArrayList<Integer> list = new ArrayList<>(20); // capacity = 20

Key distinction:
----------------
size() → number of elements currently in list
capacity → length of the internal array

How ArrayList Grows (Resizing Behavior)
---------------------------------
- When an element is added and the internal array is full, ArrayList resizes:
	- New Capacity Formula :
		newCapacity = oldCapacity + (oldCapacity/2);

- This means capacity increases by 50% each time.
- Steps:
	- Check if size < capacity → add element directly
	- If full → create a new array with 1.5× capacity
	- Copy old elements → new array
	- Add the new element
Example:
- Default capacity = 10
- Add 11th element → new capacity = 15
- Add 16th element → new capacity = 22

**Note: In Java 6-7, capacity increased by 100%, i.e., doubled.

2. LinkedList:-
=-=-=-=-=-=-=
- LinkedList is a doubly linked list implementation of the List interface.
- It allows dynamic memory allocation (no fixed size limitation).
- It maintains insertion order, allows duplicate elements, and nulls.
- Implements:
	- List<E>
	- Deque<E>
	- Cloneable
	- Serializable

- Does NOT implement RandomAccess (slow index-based access)

Internal Storage of LinkedList
------------------------------
- Internally, LinkedList uses a doubly linked list.
- Each element is stored in a Node.
- Each node contains:
	- data (element)
	- reference to next node
	- reference to previous node

Default Size and Capacity
-------------------------
- LinkedList does NOT have capacity concept like ArrayList.
- It grows dynamically as elements are added.

Key distinction:
----------------
- size() → number of elements currently in list
- capacity →  Not applicable in LinkedList

How LinkedList Grows (Memory Behavior)
--------------------------------------
- No resizing like ArrayList.
- Each element is stored in a new node.

Steps:
- Create a new node
- Link it with previous and next node
- Update pointers

Example:

Add element → new node created
No copying of old elements required

Creation of a LinkedList:-
=-=-=-=-=-=-=-=-=-=-=-=-=

1. Using default constructor without any type safety:-
2. Using default constructor with type safety:-
Example:-
	LinkedList li = new LinkedList();
	LinkedList<String> li1 = new LinkedList<>();

Methods Specific to LinkedList Class Only:-
=-=-=-=-=-=-=-=-=-=-==-=-=-=-=-=-=-=-=-=
Specific for Linked List:-
-------------------------
- void addFirst(E e) → Inserts element at the beginning.
- void addLast(E e) → Inserts element at the end.
- E getFirst() → Returns first element.
- E getLast() → Returns last element.
- E removeFirst() → Removes first element.
- E removeLast() → Removes last element.
- E push(E e) → Adds element at front.(Can help to implement Stack)
- E pop() → Removes element from front.(Can help to implement Stack)

Methods from Deque:-
-------------------
- boolean offer(E e) → Adds element at end (queue style).(Can help to implement Queue)
- boolean offerFirst(E e) → Adds element at beginning.(Can help to implement deque)
- boolean offerLast(E e) → Adds element at end.(Can help to implement deque)
- E peek() → Retrieves head without removing.
- E poll() → Retrieves and removes head. (Can help to implement Queue)

Example code:-
=-=-=-=-=-=-=
public class LinkedListExample {
    public static void main(String[] args) {
        LinkedList<String> li1 = new LinkedList<>();
        
        //Adding elements at the last Methods
        li1.add("Krish");
        li1.offer("Himanshu");
        li1.addLast("Satyam");
        li1.offerLast("Koushik");
        System.out.println(li1);

        //Adding elements at the first Methods
        li1.addFirst("Nikita");
        li1.push("Khushi");
        li1.offerFirst("Sargam");

        System.out.println(li1);

        //Other Methods

        System.out.println(li1.getFirst());
        System.out.println(li1.getLast());
        System.out.println(li1.peek());
        System.out.println(li1.removeFirst());
        System.out.println(li1.removeLast());
        System.out.println(li1.poll());

        System.out.println(li1);
    }
}

Why we can use LinkedList as Queue, Deque, and Stack Operations:-
--------------------------------------------------------
- LinkedList is a data structure that allows fast insertion/deletion at both ends, which perfectly matches the Deque contract.
- ArrayList cannot efficiently implement Deque because adding/removing from head requires shifting all elements.

Basic difference btw ArrayList and LinkedList:-
------------------------------------------------
Feature			ArrayList								LinkedList
-------                 ---------                       ----------
Data Structure		Dynamic Array						Doubly Linked List
Memory Storage		Continuous (array)					Non-contiguous (nodes)
Access Time			Fast (O(1))							Slow (O(n))
Insertion			Slow (O(n))							Fast (O(1) at ends)
Deletion			Slow (O(n))							Fast (O(1) at ends)
Memory Usage		Less (no extra pointers)			More (extra prev & next pointers)
Resizing			Required (1.5x growth)				Not required
Implements			List, RandomAccess					List, Deque


Why we need LinkedList when we have ArrayList:-
=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=
Even though ArrayList is faster in many cases, LinkedList is useful in specific scenarios :-
------------------------------------------------------------------------------------------
- Frequent Insertions/Deletions
- When Data Size Changes Frequently
- No resizing overhead (ArrayList needs copying)
- When Insertion in Middle is Frequent, Faster than ArrayList (no shifting of all elements)



Vector
=-=-=-=
- Vector is a resizable array implementation of the List interface.
- It is synchronized (thread-safe) unlike ArrayList.
- Maintains insertion order, allows duplicate elements, and nulls.
- Implements:
	- List<E>
	- Cloneable
	- Serializable

Internal Storage of Vector
---------------------------
- Internally, Vector uses an Object[] array to store elements.
- Initial capacity can grow dynamically.
- Default Size and Capacity
	- Default initial capacity = 10
		- Vector<Integer> v = new Vector<>(); // capacity = 10
	- If you provide an initial capacity:
		- Vector<Integer> v = new Vector<>(20); // capacity = 20

Key distinction:
----------------
- size() → number of elements currently in vector
- capacity → length of the internal array

How Vector Grows (Resizing Behavior)
-------------------------------------
- When elements exceed capacity, Vector doubles the capacity (100% increase).
Steps:
- Create a new array with double size
- Copy old elements → new array
- Add new element

Example:

- Default capacity = 10
- Add 11th element → new capacity = 20
- Add 21st element → new capacity = 40

Creation of a Vector
=-=-=-=-=-=-=-=-=-=

- Using default constructor:
	Vector<Integer> v = new Vector<>();

- Using constructor with initial capacity:
	Vector<Integer> v = new Vector<>(50);

- Using constructor with another collection:
	Vector<Integer> v = new Vector<>(Arrays.asList(1,2,3));

Methods Specific to Vector Class Only
=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=

- void ensureCapacity(int minCapacity) → Increases internal array capacity if needed
- void trimToSize() → Trims capacity to match current size
- synchronized methods for thread-safety, e.g., add(), remove(), get(), etc.


Stack
=-=-=
- Stack is a LIFO (Last In First Out) data structure implemented as a subclass of Vector.
- Maintains insertion order, allows duplicates and nulls.
- Implements:
	- List<E>
	- Cloneable
	- Serializable

Internal Storage of Stack
--------------------------
- Internally, Stack uses Vector for storage.
- Elements are stored in an array (dynamic).

Default Size and Capacity
-------------------------
- Inherits Vector’s default capacity = 10
- Resizes automatically when full (doubles capacity).

How Stack Works (Memory Behavior)
----------------------------------
- Stack follows LIFO (Last In First Out).
- Push → adds element at top
- Pop → removes element from top
- Peek → returns element at top without removing

Creation of a Stack
=-=-=-=-=-=-=-=-=-=
- Using default constructor:
	- Stack<Integer> s = new Stack<>();

Methods Specific to Stack Class Only
=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=
- void push(E e) - Adds element at top of stack
- E pop() - Removes and returns element from top
- E peek() - Returns element at top without removing
- boolean empty() - Returns true if stack is empty
- int search(Object o) - Returns 1-based position from top; -1 if not found


Example Code:-
------------
public class Demo {

		Stack<String> st = new Stack<String>();
		st.push("String");

		System.out.println("**********************************");
		System.out.println(st.search("Abstraction"));
		System.out.println("**********************************");
		while(!st.isEmpty()) {
			String str = st.pop();
			System.out.println(str);
			
		}
	}
}
