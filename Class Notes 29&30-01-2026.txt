		int rev = 0;
		while(num!=0) {
			int digit = num%10;
			rev = rev*10 + digit;
			num = num/10;
		}
		if(rev==dup) {
			System.out.println("Palindrome");
		}else {
			System.out.println("Not a palindrome");

Java Packages:
==============

--A package in Java is a namespace that groups related classes and interfaces in a single unit.

--A package is used to bind the related concept logically.


Purposes of Packages:

1. Logical Grouping

--Groups related classes and interfaces together.

2. Namespace Management

--Prevents naming collisions (same class name in different packages).

Example:

com.chitkara.Demo
com.masai.Demo

3. Access Control / Security

--To provide some level of security with the help of access modifiers

--Will discuss this concept in upcoming sessions when we talk about the access modifiers and inheritance concept.

Functionality of the `new` Operator
-----------------------------------
1. When the JVM encounters the `new` operator, it allocates memory for an object "inside the Heap area".

2. In that allocated memory, the JVM creates space for "all non-static members" of the class:

   - Instance variables
   - References to instance methods

3. For non-static methods:

   - Only the "method reference (name + signature + address)" is associated with the object
   - The "method body is not copied into the object"
   - Method code is executed "only when the method is called"

4. For non-static variables:

   - Memory is allocated inside the object
   - If not explicitly initialized, the JVM assigns "default values" (e.g., `0`, `null`, `false`)

5. "Local variables do not get default values":

   - They must be explicitly initialized
   - Using an uninitialized local variable causes a "compile-time error"

Note:-
----
Each object has its own separate memory:

   - Multiple objects of the same class can exist
   - Changes made to one object do "not affect other objects"
